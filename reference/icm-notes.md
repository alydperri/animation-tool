# ICM Notes

This repo is organized for the competition brief and Interpretable Context
Methodology.

## Competition Requirements Applied

- `brief.md` states the real personal/work problem in under 250 words.
- `identity.md` defines what this specialist is.
- `rules.md` defines behavior that should not drift between runs.
- `examples.md` gives copyable usage examples.
- `reference/` contains deeper workflow contracts and stage details.
- `README.md` is written for a stranger who needs to understand and use the
  folder system without hand-holding.

## ICM Principles Applied

- Put durable context in files rather than relying on chat memory.
- Separate identity, rules, examples, and references so each has one job.
- Make workflows interpretable through numbered stages and visible checkpoints.
- Keep source inputs, temporary artifacts, and approved outputs in distinct
  folders.
- Use deterministic local tools for repeatable post-processing.
- Preserve human-in-the-loop judgment at points where taste and intent matter.
