# Reference

This folder contains the stable contracts for the folder-based specialist.

- `workflow.md` explains the complete spritesheet pipeline and review gates.
- `workspace.md` explains where inputs, temporary files, and final outputs live.
- `checklists.md` gives competition, run, and repo hygiene checks.
- `icm-notes.md` summarizes the competition and Interpretable Context
  Methodology constraints used to organize this repo.
- `stages/` contains the detailed stage procedures used by agents and humans.

Use the top-level `README.md` first. Use this folder when you need the exact
stage contract or are changing the workflow itself.
